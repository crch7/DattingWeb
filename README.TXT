Additional Functionality

1. Terms of Service, Privacy Policy, and Contact Information

   - The application includes pages for Terms of Service and Privacy Policy. Contact information is stored in the database and dynamically displayed in the application.

2. Extra Photos for Each User

   - Users can upload up to 5 additional photos for other users to see them. For that, we created a new class AdditionalPhoto with a one-to-many relationship to the profile (a user can upload many photos).
   - To upload and delete them, the user can do it from "My Profile", having a button to add them and one button to delete each already uploaded photo.
   - To see pictures of other people, the user can go to the corresponding profile, where the user can see and select the photo they zoom in through some thumbnails.

3. Real Cases on the Principal Page (JavaScript Carousel)

   - The principal page features real cases that rotate automatically every 7 seconds using JavaScript. This carousel moves to the right and provides a dynamic visual experience.


4. Advanced Matching Queries

   - We have divided the functionality into two sections. The first section displays all possible matches that fall within your preferred age range and    	belong to the gender you are interested in. In the second section, we suggest profiles that have at least two hobbies in common with you and are 	also looking for the same type of relationship as you. Moreover, this section highlights profiles that share similar music tastes, personality 	traits, and cuisine preferences.

5. Table Availability

   - When proposing a date, the user must first select a day. After doing that, via JavaScript, the buttons to select the tables appear, and after selecting the table, the ones for selecting a time appear.
   - The number of available tables changes depending on the day of the week: There are 3 available tables from Monday to Thursday, 4 on Friday and Sunday, and 5 on Saturday.

6. Unblocking (clarification):
   - To unblock a person, the user can do it from the person's profile, which the user can acces from "My Profile", where the blocked users are listed. In the person's profile, there is a button that works both for blocking and unblocking (as well as one for liking and unliking).

7. JavaScript Code:
   - We implemented JavaScript code for selecting the Birth Year when creating the account, limited between 1950 and 2006 (as people under 18 should not use the app) 
   - We implemented JavaScript code to ensure the mandatory attributes in different forms are selected before creating, to avoid errors. For example, we ensured it for the birthyear when creating a profile. We also implemented code to ensure we cannot choose more than 5 hobbies.
